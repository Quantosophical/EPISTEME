Three Independent Signals

The Epistemic Decomposition

H

Entropy

Measures how uncertain the model's token distribution is. High entropy means the model is choosing between many plausible alternatives — a sign of genuine uncertainty.

H = −Σ p(t) log p(t)
G

Gradient

Measures structural importance. If negating a claim causes the entire answer to change, that claim is load-bearing. High gradient + high entropy = dangerous dependency.

G = d(A, A|¬claim)
C

Consistency

Asks the same question multiple times. If the model gives the same answer every time, it's consistent. If it contradicts itself, the claim is epistemically fragile.

C = mean(sim(sᵢ, sⱼ))
Composite Score

One number. Three signals. Complete picture.

U = H × G × (1 − C)

Uncertainty is only high when all three conditions are met: the model is uncertain (H), the claim is structurally important (G), and the model contradicts itself (1−C). Any single safe signal zeroes out the alarm.

Zone Classification

Every claim gets a verdict

■ Solid
Low uncertainty. The model is confident, consistent, and the claim isn't structurally fragile.
U ≤ 0.03
■ Gradient
Moderate uncertainty. Some signals are elevated. Worth double-checking before acting on it.
0.03 < U < 0.08
■ Fault Line
High uncertainty. The model is guessing, contradicting itself, or the claim is dangerously load-bearing.
U ≥ 0.08
Architecture

Two engines. One workspace.

KAIROS
Epistemic Uncertainty Engine

Decomposes every AI response into atomic claims and scores each one across three independent signals — entropy, gradient, and consistency. Classifies claims into SOLID, GRADIENT, or FAULT LINE zones so you know exactly what to trust.

MAKS
Adaptive Memory System

Manages long-term memory under token pressure using biologically-inspired survival scoring. Memories degrade from FULL → PARTIAL → GHOST fidelity. Ghost memories can be reconsolidated when contextually relevant — just like human memory reconsolidation.

MAKS — Memory Architecture

AI memory that forgets intelligently

Traditional chatbots dump old context. MAKS uses survival scoring to decide what to keep, what to compress, and what to archive — maintaining the most epistemically important memories under any token budget.

◈

Survival Scoring

Every memory gets a survival score based on recency, access frequency, and epistemic entropy. Memories that are recent, frequently accessed, and low-entropy survive longer. The formula mirrors biological memory consolidation.

◇

Ghost Reconsolidation

When a memory's survival drops too low, it's archived as a "ghost" — compressed but not deleted. If a new query is semantically similar to a ghost, it gets reconsolidated back into active memory. Dead knowledge can return.

▣

Token Pressure

MAKS operates under a fixed token budget. When capacity is exceeded, the lowest-survival memories are evicted first. This creates natural selection pressure — only the most relevant, certain memories persist.

▥

Entropy-Penalized Retention

Memories derived from FAULT LINE claims get a survival penalty. The system preferentially forgets uncertain information and retains knowledge it's confident about — epistemic quality shapes memory.
